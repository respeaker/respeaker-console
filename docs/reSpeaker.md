# ReSpeaker XVF3800 上位机控制软件

**Product Requirements Document（PRD）**

## 1. 项目背景（Background）

**ReSpeaker XVF3800 USB 麦克风阵列是一款集成语音前处理（AEC / Beamforming / DOA / VAD 等）的专业音频硬件，目前官方仅提供命令行或示例脚本（如 `../scripts/xvf_host.py`）进行控制，存在以下问题：

* **不适合非开发者或现场运维人员使用**
* **参数配置不直观，易误操作**
* **缺乏实时可视化能力（如 DOA / VAD 状态）**
* **不利于商业化分发和技术支持**

**本项目目标是开发一款****跨平台（macOS / Windows / Linux）的桌面上位机应用**，用于对 ReSpeaker XVF3800 设备进行**可视化配置、实时监控和维护管理**。

---

## 2. 产品目标（Goals）

### 2.1 核心目标

* 提供 **图形化、低学习成本** 的设备控制方式
* 覆盖 `xvf_host.py` 的 **全部核心控制能力**
* 满足 **商业软件分发** 的稳定性和可维护性要求

### 2.2 非目标（Out of Scope）

* **不包含音频录制、回放或波形编辑功能**
* **不作为通用声卡设置工具（OS 音频路由仍由系统负责）**
* **不实现算法本身（仅作为参数/控制面板）**

---

## 3. 用户画像（Target Users）

### 主要用户

* **嵌入式/语音算法工程师**
* **硬件调试与生产测试人员**
* **客户现场技术支持 / FAE**
* **系统集成商（智能音箱、机器人、会议系统）**

### 使用场景

* **实验室调试算法参数**
* **生产线快速验证设备状态**
* **客户现场诊断“听不见 / 方向不对 / 误唤醒”**
* **固件升级或配置恢复后的校验**

---

## 4. 系统架构概述（High-level Architecture）

### 4.1 架构形式

* **Electron 桌面应用**
* **前端：HTML / CSS / Vue**
* **后端：**
  * **USB 控制层（Node native）**
  * **设备协议封装（等价于 **`xvf_host.py`）

### 4.2 通信模型

```
UI (Renderer)
   ↓ IPC
Device Controller (Main)
   ↓ USB / localhost
ReSpeaker XVF3800
```

---

## 5. 功能需求（Functional Requirements）

### 5.1 设备管理模块（Device Management）

#### 功能点

* **自动扫描并列出所有已连接 ReSpeaker 设备，但不支持同时连接多个设备，切换设备前需要先断开当前连接**
* **显示设备信息：**
  * **产品名称**
  * **USB VID/PID**
  * **序列号（如可用）**
  * **固件版本**
  * **连接状态（已连接 / 断开）**

#### 交互要求

* **支持多设备选择（单选）**
* **设备断连时给出明确提示，不需要自动重试**

---

### 5.2 实时状态监控（Live Monitoring）

#### 功能点

* **DOA（声源方向）**
  * **数值显示（角度）**
  * **极坐标/罗盘可视化**
* **VAD（语音活动检测）**
  * **实时状态指示（ON/OFF）**
* **（可选）音频能量/状态指示**

#### 性能要求

* **刷新频率：10–30 Hz**
* **UI 不得阻塞或卡顿**

---

### 5.3 LED 控制模块（LED Control）

#### 功能点

* **LED 开关**
* **亮度调节**
* **颜色选择**
* **效果模式（如旋转、呼吸、静态）**

#### 约束

* **参数范围校验（避免非法写入）**
* **写操作需即时反馈成功/失败**

---

### 5.4 算法与音频参数配置（Audio / Algorithm Settings）

> **以设备实际支持参数为准（通过命令表驱动 UI）**

#### 可能包含

* **AEC 开关**
* **AGC 参数**
* **Noise Suppression 开关/等级**
* **Beamforming 模式**
* **Wake-word / VAD 相关阈值（如固件支持）**

#### 设计要求

* **所有参数：**
  * **显示当前值**
  * **明确单位与取值范围**
* **滑块 / 下拉框 / 输入框按类型自动适配**

---

### 5.5 配置管理（Configuration Management）

#### 功能点

* **保存当前配置到设备（Save to Flash）**
* **恢复出厂配置（如支持）**
* **重启设备**

#### 安全要求

* **危险操作需二次确认**
* **明确提示不可逆风险**

---

### 5.6 日志与诊断（Diagnostics）

#### 功能点

* **显示通信错误、超时、权限问题**
* **导出诊断信息（JSON / ZIP）：**
  * **设备信息**
  * **当前参数快照**
  * **错误日志**
